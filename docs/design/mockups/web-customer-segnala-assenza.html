Note di design. min/max del selettore data rispecchiano lato client il vincolo server (min = max(oggi, inizio abbonamento), max = fine abbonamento); il 409 "giorno già segnalato" è intercettato anche lato client prima del submit (stesso messaggio), un 422 (consenso non attivo / data fuori periodo) mostra un messaggio generico — il server resta l'autorità sulle guardie. Ownership a due assi (ADR-0049): RLS isola il tenant, actingCustomerId vincola il booking al cliente autenticato — un mismatch (booking di un altro cliente, o di un altro tenant) risponde 404, non 403 (nessun leak d'esistenza). Nessuna cassa toccata (amountCollected/refundedAmount invariati, ADR-0048). Componenti riusati dal ui-kit (Modal/Field/Input /Textarea/Button); nessun nuovo componente.
